# 시그널 품질 개선 설계 (2026-07-02)

## 배경

- 현재 활성 전략은 `breakout`(60일 신고가 돌파) 단 하나 → 알림이 항상 고점 부근 종목
  ("과매수"로 체감). 버그가 아니라 유일 활성 전략의 구조적 성격.
- 목표: (1) 진입 성격이 반대인 과매도/눌림목 전략을 검증 후 추가, (2) breakout의
  과열 진입을 데이터 근거로 걸러 승률 개선 시도, (3) 전반 결함 점검.
- 대원칙: 백테스트는 과거 성과이며 미래 수익을 보장하지 않는다. 생존 편향 고지 유지.
  검증 탈락은 정직하게 비활성 유지("개선 없음"도 결과다).

## Part C — 프로그램 전반 결함 점검 (선행)

2026-07-02 KR 스캔 크래시(0가격 봉 → ZeroDivisionError, 8a73dce로 수정)와 동류의
잠재 결함을 스캔한다:

- 0으로 나누기 / 빈 프레임 / NaN 전파 방어 누락 (fetcher → store → indicators →
  report 데이터 경계 중심)
- 발견 건은 심각도 분류 후 TDD(실패 테스트 → 수정 → 전체 스위트)로 해결

## Part A — 과매도/눌림목 전략 3종 Phase-4 검증

대상: `pullback`, `connors_rsi2`, `zscore_meanrev` (전부 현재 비활성).

1. 기존 하네스 `main.py backtest` 풀 검증 (US 80종목 게이팅 / KR 20종목 참고,
   IS/OoS·walk-forward·MC 1,000회·민감도 ±20%·레짐 슬라이스·벤치마크)
2. GateReport 판독 → **통과 전략만** `strategies.yaml` `enabled: true` + 검증
   성적 주석 (breakout 스타일)
3. 경계선·탈락 → 비활성 유지, 사유 보고
4. 활성 상한 7개 확인 (현재 1 → 최대 4)

## Part B — breakout 과열 가드 (사전등록 기준 채택)

1. breakout에 **선택적** 가드 파라미터 3종 구현 — YAML 미설정 시 기존과 동일
   동작(베이스라인 보존 불변식):
   - `max_ext_atr`: (종가−SMA20)/ATR ≤ k — 변동성 보정 이격도
   - `max_ext_pct`: 종가/SMA20 − 1 ≤ X%
   - `rsi_max`: RSI(14) ≤ 상한
2. 베이스라인 vs 가드 소규모 그리드(유형 3 × 값 2~3 — 과적합 방지) IS/OoS 비교
3. **채택 기준 (사전등록, 변경 금지)**: OoS 승률 ≥ 베이스라인 +3%p
   **AND** OoS PF ≥ 베이스라인 × 0.9 **AND** OoS 거래수 ≥ 베이스라인 × 0.7.
   미달 시 가드 미채택.
4. 채택 시 breakout은 새 파라미터로 Phase-4 게이트 **재통과 후에만** YAML 반영.

## 실행 순서

C(빠른 안전점검) → A(풀 검증, 소요 큼) → B(가드 구현·비교). B는 A와 검증 하네스를
공유하므로 이 순서가 실행 횟수 최소.

## 테스트 전략
