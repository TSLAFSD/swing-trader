# 📈 swing-trader

완전 서버리스 · 무료 스윙 트레이딩 분석 파이프라인.
GitHub Actions가 정해진 시간에 한국·미국 주식 전체를 스캔하고, 매수 후보를 **텔레그램**으로 보내며, 상세 분석은 **GitHub Pages 리포트** 링크로 확인합니다.

> ⚠️ **이 시스템은 추천만 합니다. 매매 결정과 책임은 전적으로 본인에게 있습니다.**
> 자동 주문 기능은 없으며 의도적으로 만들지 않았습니다.

## 어떻게 동작하나

| 시간 (KST) | 무슨 일이 일어나나 |
|---|---|
| 평일 12:37 | 🇰🇷 한국 예비 스캔 — "예비(미확정)" 태그로 참고용 알림 |
| 평일 15:47 | 🇰🇷 한국 확정 스캔 — 매수 후보 + 보유 종목 매도 점검 |
| 화–토 07:07 | 🇺🇸 미국 확정 스캔 (러셀 1000) |
| 평일 16:37 | 🇺🇸 프리마켓 갭 체크 — 아침 시그널 가격이 크게 변했는지 확인 |
| 일요일 04:07 | 📊 주간 점검 — 시그널 적중률 리포트, 전략 재검증 |

GitHub Actions의 cron은 혼잡 시간대에 15~30분 지연될 수 있습니다 — 설계상 문제없습니다.

## 전략 활성화 원칙

6개 전략(눌림목, Z-Score 평균회귀, 단기 과매도 반등, 돌파, 변동성 스퀴즈, 와이코프 스프링)이 구현되어 있지만, **6단계 검증 게이트(표본외 수익성·워크포워드·몬테카를로·민감도 등)를 통과한 전략만 활성화**됩니다. 현재 활성: **돌파(breakout)** 1개. 나머지는 게이트 미달로 비활성 — 주간 재검증에서 추적됩니다.

또한 발송 단계에 별도 컷오프가 있습니다: 종목별 과거 성과 PF < 1.0, 표본 < 5건,
종합 강도 < 20, 손절폭 > 15% 시그널은 **리포트는 생성되되 텔레그램으로 발송되지
않으며**, 헬스체크에 "(필터 제외 N건)"으로 표기됩니다.

## ⚠️ 정직 고지 (반드시 읽어주세요)

- 모든 백테스트 확률은 **과거 데이터** 기준이며, 실전 성과는 일반적으로 그보다 낮습니다.
- 백테스트는 **현재 상장 종목** 기준입니다. 상장폐지된 종목이 빠져 있어 승률·수익률이 **낙관적으로 왜곡**(생존 편향)되어 있습니다.
- 본 시스템의 어떤 출력도 투자 자문이 아닙니다.

## 리포트 화면 안내 (U5/U6 업그레이드)

- **차트**: TradingView lightweight-charts 기반. 핀치로 확대, 더블탭으로 리셋,
  상단 버튼으로 **일봉↔주봉** 전환과 **1M/3M/6M/1Y/전체** 기간 선택이 됩니다.
  캔들 위 가로선은 제안 매수가(파란 실선)·매수 범위 상단(점선)·제안 손절(빨강)입니다.
  맨 아래 패인은 Weis Wave — 파동 방향별 누적 거래량으로 세력의 '노력'을 보여줍니다.
- **🧭 Wyckoff VPA 진단**: 유동성 저점 → 셀링 클라이맥스 → 공급 고갈 3단계 체크.
  3단계 모두 충족이면 텔레그램 카드에 🟢 매집권 배지가 붙습니다.
- **📋 AI 분석 요청 복사** (하단 고정 버튼): 누르면 차트 데이터·와이코프 진단·지표가
  담긴 분석 요청문이 복사됩니다. Claude 등 AI 채팅에 붙여넣으면 단계별 검증 분석을
  받을 수 있습니다. (자동 호출이 아닌 수동 복사 — 비용·노이즈 차단을 위한 의도된 설계)
- **종합 등급 A/B/C**: 강도×0.5 + 신뢰도×0.3 + 시장국면×0.2. 산출 근거가 리포트
  상단에 그대로 표기됩니다.

## 포지션 슬롯

동시 보유는 **5슬롯**(`MAX_POSITION_SLOTS`) 기준으로 안내됩니다. 보유 요약에
"📊 3/5 슬롯"과 "남은 슬롯 N — TOP N만 선별 권장"이 표시되고, 만석 상태에서 새
시그널이 오면 "⚠️ 슬롯 가득" 태그가 붙습니다 (차단이 아닌 안내).
ATR 추적 손절 포지션은 매 확정 스캔 후 `positions.yaml`에 최고 종가와 현재
추적 손절가가 자동 기록됩니다 (값이 변한 날만 커밋).

## 일상 사용법

1. **알림 받기**: 스캔 시간마다 텔레그램으로 결과가 옵니다. 시그널이 없어도 "✅ 스캔 완료 · 시그널 0개" 헬스체크가 옵니다 — **알림 자체가 안 오면 시스템 고장**이니 GitHub Actions 탭을 확인하세요.
2. **리포트 보기**: 메시지의 📄 링크를 누르면 Safari에서 차트·지표 해설·매매 계획이 열립니다.
3. **매수했다면**: `/add AAPL 230.5 10` (티커 가격 수량) — 다음 스캔부터 손절/목표 도달 시 매도 알림이 옵니다.
4. **매도했다면**: `/remove AAPL` — 종목이 다시 시그널 대상으로 돌아갑니다.
5. **궁금한 종목**: `/analyze 005930` — 시그널 여부와 무관하게 전체 진단 리포트를 받습니다.
6. **오류 알림이 오면**: 메시지의 코드 블록 전체를 복사해 Claude Code에 붙여넣고 "고쳐줘"라고 하면 됩니다.

> ⚠️ **명령은 한 번에 하나씩 보내세요.** 스캔이 돌아가는 중에 명령을 연달아 보내면
> GitHub의 대기열 규칙(그룹당 대기 1개)에 따라 마지막 명령만 실행되고 앞의 것은
> 조용히 취소됩니다. ⏳ 접수 후 5분 넘게 결과가 없으면 같은 명령을 다시 보내세요.

수동 실행 (터미널):

## 📚 지표 입문 해설 (차트를 모르는 분을 위해)

리포트에 나오는 용어들입니다. 전부 외울 필요 없이, 필요할 때 찾아보세요.

- **이동평균선 (SMA)**: 최근 N일 종가의 평균. "주가가 200일선 위"면 장기 상승 추세라는 뜻입니다. 20일선은 단기, 60일선은 중기, 200일선은 장기 기준선입니다.
- **RSI**: 0~100 사이의 "과열/침체 온도계". 30 아래면 과매도(많이 떨어짐), 70 위면 과열(많이 오름). RSI(2)는 2일짜리 초단기 버전이라 훨씬 민감합니다.
- **z-score (표준편차 점수)**: 주가가 20일 평균에서 얼마나 벗어났는지를 통계 단위로 잰 값. **−2.5 이하면 통계적으로 드문 수준의 급락** — 평균으로 되돌아올 확률에 베팅하는 근거가 됩니다. +2.5 이상이면 반대로 과열입니다.
- **볼린저밴드**: 20일 평균 주위로 "주가가 보통 머무는 범위"를 그린 띠. 하단 근처면 싸게, 상단 근처면 비싸게 거래되는 중입니다.
- **ADX**: 추세의 **세기**(방향 아님)를 재는 지표. 20 이상이면 추세가 형성됐다고 봅니다.
- **MACD**: 단기 평균과 장기 평균의 간격으로 모멘텀 방향을 보는 지표. 히스토그램이 0 위면 매수 우위입니다.
- **ATR**: 하루 평균 변동폭. "ATR 추적 손절"은 최고가에서 ATR의 3배만큼 내려오면 파는 방식 — 이익은 달리게 두고 하락만 자르는 장치입니다.
- **거래량 배수**: 평소(20일 평균) 대비 오늘 거래량. 돌파나 반등에 거래량이 실리면 신뢰도가 높아집니다.
- **승률 / PF(Profit Factor)**: 과거 같은 신호로 샀을 때 이긴 비율 / 총이익÷총손실. **PF 1.0이 손익분기**이며, 표본이 10건 미만이면 수치를 믿지 마세요 (리포트가 경고를 띄웁니다).
- **MDD (최대 낙폭)**: 과거 최악의 순간에 자산이 고점 대비 몇 % 빠졌었는지.
- **하프 켈리**: 수학적 최적 베팅 비율(켈리 공식)의 절반. 참고용 제안일 뿐 지시가 아닙니다.
- **시장 폭 (breadth)**: 전체 종목 중 60일선 위에 있는 종목의 비율. 지수는 대형주 몇 개로 왜곡될 수 있어, 시장 내부 체력은 이 수치로 봅니다.
- **VIX**: 미국 시장의 "공포 지수". 20 이하면 안정, 30 이상이면 불안 국면입니다 (참고용 표시).

## 구조 (개발자용)

```
config/    설정·전략 파라미터(YAML)·보유 종목
src/data      수집(미국 yfinance / 한국 FDR→pykrx→yfinance 3중) + Parquet/DuckDB 저장
src/analysis  지표·7전략·국면 필터·시그널 엔진
src/risk      청산 엔진·켈리·상관·서킷브레이커·갭 가드
src/backtest  검증 게이트·종목별 신뢰도·실전 추적
src/report    Plotly 차트·HTML 리포트·Pages 배포
src/notify    텔레그램 발송·메시지 포맷
worker/       Cloudflare Worker (텔레그램 명령 수신)
```

데이터는 orphan `data` 브랜치에 단일 squash 커밋으로 보관(영구 누적), 리포트는 `gh-pages` 브랜치로 서빙됩니다.
